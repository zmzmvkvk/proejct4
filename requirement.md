# AI 쇼츠 자동화 프로그램 요건 정의

## 요건 정의를 위한 질문 목록

### 1. 기본 기능 요구사항

1. **스토리 작성 및 관리**

   - 사용자가 스토리를 작성할 수 있는 텍스트 에디터가 필요한가요?
     - 장면 구분자('---')를 사용한 멀티 씬 스토리 작성 지원
   - 스토리 템플릿 또는 예제 제공이 필요한가요?
     - 사이버펑크, 판타지 등 다양한 장르별 예제 제공
   - 스토리 저장 및 불러오기 기능이 필요한가요?
     - 프로젝트 단위로 스토리 관리

2. **캐릭터 및 에셋 관리**

   - 캐릭터 생성 및 관리 기능이 필요한가요?
     - 캐릭터 이름, 참조 이미지, 설명 등 관리
   - 커스텀 에셋 학습 기능이 필요한가요?
     - Leonardo.ai의 Custom Model 학습 기능 활용
   - 에셋 상태 추적 및 관리가 필요한가요?
     - 학습 진행 상태, 완료 상태 등 실시간 모니터링
   - 에셋 재사용 및 공유 기능이 필요한가요?
     - 프로젝트 간 에셋 공유, 에셋 라이브러리 관리

3. **AI 이미지 생성 기능**
   - 어떤 AI 이미지 생성 모델을 사용할 예정인가요?
     - Leonardo.ai API 사용
   - 프롬프트 최적화 기능이 필요한가요?
     - OpenAI GPT를 통한 프롬프트 강화 및 최적화
   - 배치 생성 기능이 필요한가요?
     - 여러 장면을 한 번에 생성하는 기능
   - 이미지 후처리 기능이 필요한가요?
     - 기본적인 크롭, 리사이즈 등

### 2. 사용자 인터페이스 요구사항

1. **레이아웃 및 디자인**

   - 대시보드 형태의 레이아웃이 필요한가요?
     - 프로젝트 선택, 스토리 작성, 스토리보드 뷰어 등 통합 인터페이스
   - 다크 모드/라이트 모드 지원이 필요한가요?
     - 필요함
   - 반응형 디자인이 필요한가요?
     - 필요함 (데스크톱 우선, 모바일 지원)

2. **사용자 경험**
   - 드래그 앤 드롭 기능이 필요한가요?
     - 이미지 업로드, 씬 순서 변경 등
   - 실시간 미리보기 기능이 필요한가요?
     - 스토리보드 실시간 업데이트
   - 진행 상황 표시 기능이 필요한가요?
     - 이미지 생성 진행률, 에셋 학습 진행률 등
   - 오류 처리 및 사용자 피드백이 필요한가요?
     - 토스트 메시지, 에러 모달 등

### 3. 기술적 요구사항

1. **프론트엔드 기술**

   - React 18+ 사용
   - Vite 빌드 시스템 사용
   - Zustand를 통한 상태 관리
   - 컴포넌트 기반 아키텍처

2. **백엔드 기술**

   - Node.js + Express 서버
   - RESTful API 설계
   - 외부 API 통합 (Leonardo.ai, OpenAI)
   - Firebase/Firestore 데이터베이스

3. **외부 서비스 통합**
   - Leonardo.ai API 연동이 필요한가요?
     - 이미지 생성, 커스텀 모델 학습, 에셋 관리
   - OpenAI API 연동이 필요한가요?
     - 프롬프트 강화, 스토리 분석
   - 클라우드 스토리지 연동이 필요한가요?
     - Firebase Storage 또는 AWS S3

### 4. 데이터 관리 및 보안

1. **데이터 저장**

   - 프로젝트 데이터를 어떻게 저장할 예정인가요?
     - Firestore를 통한 클라우드 저장
   - 이미지 파일을 어떻게 관리할 예정인가요?
     - 외부 URL 참조 또는 클라우드 스토리지
   - 사용자 데이터 백업이 필요한가요?
     - 자동 백업 및 복원 기능

2. **보안**
   - API 키 보안 관리가 필요한가요?
     - 환경 변수를 통한 서버 사이드 키 관리
   - 사용자 인증이 필요한가요?
     - 현재는 불필요, 추후 확장 가능
   - 데이터 암호화가 필요한가요?
     - 기본적인 전송 암호화 (HTTPS)

### 5. 성능 및 확장성

1. **성능 최적화**

   - 이미지 로딩 최적화가 필요한가요?
     - 지연 로딩, 이미지 압축 등
   - API 호출 최적화가 필요한가요?
     - 캐싱, 요청 제한, 재시도 로직
   - 대용량 프로젝트 처리가 필요한가요?
     - 페이지네이션, 가상 스크롤링

2. **확장성**
   - 다중 사용자 지원이 필요한가요?
     - 추후 확장 가능하도록 설계
   - 다국어 지원이 필요한가요?
     - 현재는 한국어, 추후 확장 가능
   - 플러그인 시스템이 필요한가요?
     - 추가 AI 모델 연동 가능하도록 설계

### 6. 배포 및 운영

1. **배포 환경**

   - 어떤 방식으로 배포할 예정인가요?
     - 클라이언트: Vercel 또는 Netlify
     - 서버: Railway, Heroku 또는 AWS
   - 도메인 및 SSL 인증서가 필요한가요?
     - 프로덕션 환경에서 필요
   - 모니터링 및 로깅이 필요한가요?
     - 기본적인 에러 로깅 및 사용량 모니터링

2. **유지보수**
   - 자동 배포 파이프라인이 필요한가요?
     - GitHub Actions를 통한 CI/CD
   - 버전 관리 전략은 어떻게 할 예정인가요?
     - Semantic Versioning 사용
   - 데이터베이스 마이그레이션 전략이 필요한가요?
     - Firestore는 스키마리스이므로 점진적 마이그레이션

### 7. 최종 선택된 기술 스택

AI 쇼츠 자동화 프로그램 개발을 위한 최적의 기술 스택은 다음과 같습니다:

1. **프론트엔드**

   - **핵심 프레임워크**: React 18
   - **빌드 시스템**: Vite
   - **상태 관리**: Zustand
   - **UI 라이브러리**: Custom CSS + Tailwind CSS
   - **HTTP 클라이언트**: Axios
   - **유틸리티**:
     - 토스트 알림 시스템
     - 디바운스 훅
     - 에러 바운더리

2. **백엔드**

   - **런타임**: Node.js 18+
   - **웹 프레임워크**: Express.js
   - **데이터베이스**: Firebase Firestore
   - **파일 스토리지**: Firebase Storage
   - **로깅**: Winston
   - **미들웨어**:
     - CORS 처리
     - 에러 핸들링
     - 요청 로깅

3. **외부 서비스 통합**

   - **AI 이미지 생성**: Leonardo.ai API
   - **AI 텍스트 처리**: OpenAI GPT API
   - **클라우드 인프라**: Firebase

4. **개발 도구**

   - **패키지 관리**: npm
   - **코드 품질**: ESLint + Prettier
   - **버전 관리**: Git + GitHub
   - **CI/CD**: GitHub Actions

5. **프로젝트 구조**
   ```
   project/
   ├── client/                 # React 프론트엔드
   │   ├── src/
   │   │   ├── components/     # UI 컴포넌트
   │   │   ├── services/       # API 서비스
   │   │   ├── stores/         # Zustand 스토어
   │   │   ├── hooks/          # 커스텀 훅
   │   │   └── utils/          # 유틸리티 함수
   │   └── public/
   ├── server/                 # Node.js 백엔드
   │   ├── config/            # 설정 파일
   │   ├── middleware/        # Express 미들웨어
   │   ├── routes/            # API 라우트
   │   ├── services/          # 외부 서비스 연동
   │   └── index.js           # 서버 진입점
   └── docs/                  # 프로젝트 문서
   ```

### 8. 핵심 기능 명세

1. **프로젝트 관리**

   - 프로젝트 생성, 수정, 삭제
   - 프로젝트 목록 조회 및 검색
   - 프로젝트별 메타데이터 관리

2. **스토리 관리**

   - 멀티 씬 스토리 작성 및 편집
   - 실시간 씬 파싱 및 미리보기
   - 스토리 템플릿 제공

3. **에셋 관리**

   - 캐릭터 에셋 생성 및 관리
   - Leonardo.ai 커스텀 모델 학습
   - 에셋 상태 추적 및 모니터링
   - 에셋 재사용 및 참조 관리

4. **이미지 생성**

   - Leonardo.ai를 통한 고품질 이미지 생성
   - 프롬프트 최적화 및 강화
   - 배치 생성 및 진행 상황 추적
   - 생성 결과 관리 및 미리보기

5. **스토리보드**
   - 실시간 스토리보드 업데이트
   - 씬별 이미지 생성 및 관리
   - 스토리보드 내보내기 및 공유

이 기술 스택과 기능 명세를 통해 확장 가능하고 유지보수가 용이한 AI 쇼츠 자동화 프로그램을 개발할 수 있습니다.
